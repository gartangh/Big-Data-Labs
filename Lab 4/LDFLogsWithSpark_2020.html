Lab Session: LDF Log Analysis with Spark and Dataframes¶

Introduction¶

One of the more promising research directions at which the Semantic Web group (headed by Erik Mannens) is excelling is the research to linked data fragments. Semantic Data is traditionally published on the web using SPARQL endpoints, which are APIs which allow you to send arbitrarily complex queries (SPARQL is very similar to SQL).

The most common SPARQL endpoint is: https://dbpedia.org/sparql

Web APIs are much more conservative but at the same time also it is much easier to guarantee their uptime. SPARQL endpoints do not restrict the end-user making their load unpredictable. It is therefore very hard to guarantee that they won't crash. This problem is addressed by our lab using the Linked Data Fragments approach (Ruben Verborgh). In this innovative approach LDF servers can only answer very simpele queries while the LDF client does the hard working of doing the joins locally. This way the effort gets balanced and uptime can be guaranteed.

  • More info: LDF website

  • More information as well in the course Internet Technology

  • Note that there are many thesis subjects on this topic as well!

The LDF concept is supposed to address the lack of high availability of SPARQL endpoints on the web. We tested this claim by putting an LDF server on Amazon EC2 and hosting a number of linked open datasets. The server logs where stored and are offered here as a dataset for you to explore. Log analysis is a Big Data use case: logs can be very high in volume but it's not entirely clear up front which information in the logs is valuable to use (schema on write), therefore collecting the logs in for example HDFS is a more sensible approach.

The safest way to parse a string is to use regular expressions, since they allow for fine-grained data quality control.

For a quick intro into regular expressions see: https://developers.google.com/edu/python/regular-expressions

For a regex tester: http://pythex.org/

Let's start with some basic symbols:

^,$ are beginning/end of string

\S matches any non-whitespace character.

\w matches a word character while \W matches any non word character

\d matches a digit

. is a wildcard (i.e. matches any character)

if you want to match special characters you need to escape them, for example \\[

Quantities:

+ 1 or more occurrences of the pattern to its left, e.g. 'i+' = one or more i's

* 0 or more occurrences of the pattern to its left

? match 0 or 1 occurrences of the pattern to its left

Brackets:

are in principle an or operator: [ab] matches a or b

curly bracks are used for repeats: \d{3} corresponds to 3 digits

def qualityCheck1(regex, logline):
    match = re.search(regex, logline)
    
    if (match == None):
        return (0, logline)
    
    else:
        return (1, Row(
        client_ip = match.group(1),
        identd = match.group(2),
        userid = match.group(3),
        server_local_time = match.group(4),
        uri_request = match.group(5),
        server_http_statuscode = match.group(6),
        server_response_size = match.group(7),
        client_referer_header = match.group(8),
        client_user_agent_header = match.group(9),
        client_accept_header = match.group(10),
        server_cache_status = match.group(11)
        ))

# convert spo string to a template such as: ***, s**, *p*, **o, sp*, ..., spo.
def convertToTripleTemplate(req):
    pattern = ""
    
    if "subject=" in req:
        pattern += "s"
    else:
        pattern += "*"
        
    if "predicate=" in req:
        pattern += "p"
    else:
        pattern += "*"
    
    if "object=" in req:
        pattern += "o"
    else:
        pattern += "*"
    
    return pattern

test = "GET /2014/en?object=%2217842723%22%5E%5Ehttp%3A%2F%2Fwww.w3.org%2F2001%2FXMLSchema%23integer HTTP/1.1"

def extractDayMonthYear(slt):
    # keep in mind that the mapping dictionary should be shipped to the workers
    slt_split = slt.split("/")
    day = int(slt_split[0].strip('['))
    month = mapping.get(slt_split[1])
    year  = int(slt_split[2].split(":")[0])
    return (day,month,year)
